# Diagrama de Classes no padrão UML

## Introdução ao Modelo de Classes

Este documento apresenta o Diagrama de Classes UML (Linguagem de Modelagem Unificada), que serve como um mapa estrutural do nosso sistema de pedidos.

O Diagrama de Classes traduz os requisitos de negócio e a estrutura do banco de dados (o Esquema Relacional) em um modelo orientado a objetos. Ele é a peça central da nossa arquitetura de software, pois define:

Quais são as Entidades (Classes) principais do sistema (como Pessoa, Pedido, Restaurante e Produto).

Quais informações (Atributos) cada entidade carrega, como o nome, o CPF ou o valor de um pedido.

Como essas entidades se conectam (Associações), ilustrando a lógica do sistema, como o fato de uma Pessoa poder realizar vários Pedidos ou um Restaurante empregar vários Entregadores.

Em essência, este diagrama é o projeto que guiará o desenvolvimento do software, garantindo que a estrutura de dados e as relações entre os componentes do sistema sejam consistentes e facilmente compreendidas pela equipe de desenvolvimento.
